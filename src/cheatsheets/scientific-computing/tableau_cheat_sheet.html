Extract vs Live Connection

Live Connection:
• Real-time data from source
• Always up-to-date
• Depends on source performance
Extract Benefits:
• Faster performance
• Offline capability
• Data optimization
• Incremental refresh

Join Types

Inner Join Records matching in both tables
Left Join All from left + matching from right
Right Join All from right + matching from left
Full Outer Join All records from both tables

Calculated Fields

SUM([Sales]) Sum aggregation
AVG([Profit]) Average aggregation
COUNT([Orders]) Count of records
IF [Sales] > 1000 THEN "High" ELSE "Low" END Conditional logic
DATEDIFF('day', [Order Date], TODAY()) Date difference calculation
LEFT([Customer Name], 1) String manipulation

Table Calculations

RUNNING_SUM(SUM([Sales])) Running total
WINDOW_AVG(SUM([Sales])) Moving average
RANK(SUM([Sales])) Rank values
PERCENT_OF_TOTAL(SUM([Sales])) Percentage of total
PREVIOUS_VALUE(0) + SUM([Sales]) Previous value reference

Level of Detail (LOD) Expressions

{FIXED [Region] : SUM([Sales])} Fixed LOD - ignores view filters
{INCLUDE [Category] : AVG([Sales])} Include LOD - adds dimension
{EXCLUDE [Sub-Category] : SUM([Sales])} Exclude LOD - removes dimension
Common LOD Patterns:
• Customer acquisition date: {FIXED [Customer] : MIN([Order Date])}
• Percentage of total: SUM([Sales]) / {FIXED : SUM([Sales])}
• Cohort analysis: {FIXED [Customer] : MIN([Order Date])}

Parameters

Creating Parameters:
• Right-click Data pane → Create Parameter
• Configure name, data type, values
• Show parameter control for user input
Parameter Uses:
• Dynamic filtering: RANK(SUM([Sales])) <= [Top N Parameter]
• What-if analysis: [Sales] * (1 + [Growth Rate Parameter])
• Dynamic measures: CASE [Measure Parameter]

Sets

Set Types:
• Manual selection (General tab)
• Formula-based (Condition tab)
• Top/Bottom N (Top tab)
IF [Customer Set] THEN "In Set" ELSE "Out" END Using sets in calculations
Combined Sets:
• Intersection (AND): Members in both
• Union (OR): Members in either
• Difference (NOT): Members in first only

Performance Best Practices

Data Level:
• Use extracts for better performance
• Filter at data source level
• Aggregate data appropriately
• Use proper data types
Calculation Level:
• Use basic calculations over table calculations
• Avoid nested calculations
• Use context filters strategically
• Minimize number of marks
Dashboard Level:
• Limit number of sheets
• Use containers efficiently
• Optimize images and logos
• Test with realistic data volumes

Design Principles

Visual Design:
• Choose right chart type for data
• Use consistent colors and formatting
• Minimize chart clutter
• Direct attention with color and size
• Make text readable
Dashboard Layout:
• Follow visual hierarchy principles
• Group related information
• Use white space effectively
• Design for target screen size
• Test on different devices
Interactivity:
• Make interactions intuitive
• Provide clear feedback
• Don't over-filter
• Include reset/clear options
• Use progressive disclosure

Troubleshooting

Data Issues:
• Null values: Check data source, use IFNULL()
• Unexpected aggregation: Check field types
• Missing marks: Check filters and densification
• Incorrect joins: Verify join conditions
Performance Issues:
• Slow loading: Check data volume
• Complex calculations: Simplify where possible
• Too many marks: Add filters or aggregation
• Use Performance Recording tool
