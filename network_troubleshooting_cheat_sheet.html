Network Troubleshooting Cheat Sheet

Essential commands and methodologies for diagnosing and fixing network connectivity issues

Basic Connectivity Tests

Ping Commands

ping 8.8.8.8 Test internet connectivity
ping -t 8.8.8.8 Continuous ping (Windows)
ping -c 4 8.8.8.8 Ping 4 times (Linux/Mac)
ping gateway_ip Test local network
ping localhost Test TCP/IP stack
ping -f -s 1472 target Test MTU size

Traceroute

tracert google.com Trace route (Windows)
traceroute google.com Trace route (Linux/Mac)
mtr google.com Continuous traceroute
pathping target Combined ping/tracert

DNS Troubleshooting

DNS Lookup Commands

nslookup domain.com Basic DNS lookup
nslookup domain.com 8.8.8.8 Query specific DNS server
dig domain.com Detailed DNS info (Linux/Mac)
dig @8.8.8.8 domain.com Query specific server
dig domain.com MX Query MX records
host domain.com Simple DNS lookup

DNS Cache

ipconfig /flushdns Clear DNS cache (Windows)
sudo dscacheutil -flushcache Clear DNS cache (Mac)
sudo systemctl restart systemd-resolved Restart DNS (Ubuntu)
ipconfig /displaydns Display DNS cache

Network Configuration

IP Configuration

ipconfig Basic IP info (Windows)
ipconfig /all Detailed IP info (Windows)
ifconfig IP configuration (Linux/Mac)
ip addr show Show IP addresses (Linux)
ip route show Show routing table
route print Show routes (Windows)
netstat -rn Show routing table

DHCP Operations

ipconfig /release Release IP address
ipconfig /renew Renew IP address
sudo dhclient -r Release DHCP (Linux)
sudo dhclient Request new DHCP

Port & Connection Testing

Port Connectivity

telnet host 80 Test port connectivity
nc -zv host 80 Test port (netcat)
nmap -p 80 host Scan specific port
nmap -p 1-1000 host Scan port range
ss -tulpn Show listening ports
netstat -an Show all connections
netstat -b Show process names

Connection Analysis

netstat -anp | grep :80 Check specific port usage
lsof -i :80 See what's using port 80
ss -tulpn | grep :80 Modern netstat alternative

Common Issue Workflows

No Internet Connection

  1. ping 127.0.0.1 (test local stack)
  2. ping gateway_ip (test local network)
  3. ping 8.8.8.8 (test internet)
  4. nslookup google.com (test DNS)
  5. ipconfig /release && ipconfig /renew
  6. ipconfig /flushdns
  7. Check cables and WiFi connection

Slow Network Performance

  1. ping -t gateway (check latency)
  2. tracert target (find bottlenecks)
  3. iperf3 -c server (bandwidth test)
  4. netstat -an (check connections)
  5. Check for duplicate IP addresses
  6. Monitor interface errors
  7. Check for network congestion

DNS Resolution Issues

  1. nslookup domain.com
  2. Try different DNS servers
  3. ipconfig /flushdns
  4. Check hosts file
  5. Verify DNS settings
  6. Test with dig/nslookup
  7. Check firewall DNS rules

Port Connectivity Issues

  1. telnet host port
  2. nmap -p port host
  3. Check if service is running
  4. Verify firewall rules
  5. Check routing tables
  6. Test from different locations
  7. Capture packets with tcpdump

Critical Troubleshooting Tips

Layer-by-Layer Approach: Always start with physical layer (cables, WiFi) and work up through OSI layers
Document Everything: Keep logs of commands and results for complex issues
Isolate Variables: Change one thing at a time to identify root cause
Baseline Knowledge: Know your normal network configuration before problems occur
Multiple Tests: Don't rely on single test results - verify with multiple methods
Time-based Issues: Some problems are intermittent - monitor over time
